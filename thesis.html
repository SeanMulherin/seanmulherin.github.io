Data Management Plan
The following is a data management plan for my Master’s thesis. The purpose of this plan is to set consistent and reliable standards to govern my data organization. Moreover, it is meant to instill transparency for data sharing purposes. My files will be locally organized using the following hierarchy:
└── User Directory/
    └── Master's Thesis/
        ├── Code: contains all .R, .RMD, .IPYNB code files
        ├── Data: contains all .CSV, .XLSX, .TXT data files
        ├── Papers: contains all downloaded articles and books
        ├── Plots: contains all figures, diagrams, and plots
        └── Masters Thesis Draft: saved pdf of most current version

All local files are automatically backed up on my personal iCloud account. File names will follow a hybrid naming convention (i.e. camelCase and pot_hole). Simple descriptive names will be used with version identifiers whenever relevant. File copies will be saved offsite on my personal Google ecosystem, Github, and Overleaf accounts, including: Google Drive, Google Docs, Google Colab, Google Sheets, Github, and Overleaf. All pertinent files will be publicly shared through my personal Github. All files will be stored both locally and on my GitHub indefinitely and for a minimum of 10 years. With my permission, the anonymized data can be accessed here.
